# Data Model: Al-Nour Eye Hospital Management System

**Feature**: 001-eye-hospital-hms
**Phase**: 1 — Design
**Date**: 2026-04-14

All primary keys are ULID strings unless noted. All tables use `created_at` /
`updated_at` timestamps (Laravel convention). Foreign key constraints enforced at DB level.

---

## Module: Booking

### Table: `bookings`

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| `id` | char(26) | PK | ULID |
| `file_no` | varchar(20) | UNIQUE, NOT NULL | MRN-YYYY-XXXXX |
| `patient_name` | varchar(150) | NOT NULL | |
| `patient_phone` | varchar(20) | nullable | |
| `patient_age` | tinyint unsigned | nullable | |
| `national_id` | varchar(20) | nullable | |
| `gender` | enum(male,female) | nullable | |
| `dept` | enum(clinic,labs,surgery,lasik,laser) | NOT NULL | |
| `service_id` | char(26) | FK→services.id, nullable | |
| `service_name` | varchar(200) | NOT NULL | Snapshot at booking time |
| `doctor_id` | char(26) | FK→doctors.id, nullable | |
| `date` | date | NOT NULL | |
| `time` | time | nullable | |
| `price` | decimal(10,2) | DEFAULT 0 | |
| `discount` | decimal(10,2) | DEFAULT 0 | |
| `ins_amount` | decimal(10,2) | DEFAULT 0 | Insurance covers this amount |
| `ins_company_id` | char(26) | FK→insurance_companies.id, nullable | |
| `pay_method` | enum(cash,card,transfer,insurance) | NOT NULL, DEFAULT cash | |
| `pay_status` | enum(unpaid,partial,paid) | NOT NULL, DEFAULT unpaid | |
| `status` | enum(confirmed,in_progress,completed,cancelled,waiting) | NOT NULL, DEFAULT waiting | |
| `visit_note` | text | nullable | |
| `cancel_reason` | text | nullable | Required on cancellation |
| `created_by` | bigint unsigned | FK→users.id | |

**Indexes**: `(date, dept)`, `(doctor_id, date)`, `(file_no)`, `(status, date)`

**Business rules**:
- `file_no` auto-generated by `MrnGeneratorService` on create
- `service_name` and `price` are snapshotted from `services` table at booking time
- `pay_status = paid` blocks hard-delete; cancellation required instead
- `cancel_reason` MUST be present when `status = cancelled`

**State machine**: `waiting → confirmed → in_progress → completed`; any state → `cancelled`

## State Machines

### Booking Status
```
waiting ──→ confirmed ──→ in_progress ──→ completed
    │            │               │
    └──────→ cancelled ←─────────┘
```

### Surgery Status
```
scheduled ──→ prep ──→ in_progress ──→ completed
                                  │
                              cancelled
```

### Doctor Shift Status
```
open ──→ closed ──→ handed_over
```

### Pay Status (Booking)
```
unpaid ──→ partial ──→ paid
```
